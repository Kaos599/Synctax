## 2026-04-24 - Predictable temporary file paths in atomic write operations\n**Vulnerability:** Weak random number generation (Math.random()) was used to construct suffixes for temporary files during atomic file writes (e.g., config updates and backup archives), leading to predictable temporary file names.\n**Learning:** In POSIX environments, even a temporary file intended to be quickly renamed can be hijacked or overwritten via symlink attacks or race conditions if its path is completely predictable. This is especially risky in multi-user systems where `os.tmpdir()` or shared directories are used.\n**Prevention:** Always use cryptographically secure random number generators (like `crypto.randomBytes(8).toString('hex')` or `crypto.randomUUID()`) when generating unique names for temporary paths.
