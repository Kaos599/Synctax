## 2024-05-18 - CLI Adapter Detection Bottleneck
**Learning:** Initializing/validating commands loop over available config adapters (like cursor, claude, etc.) and perform detection tasks sequentially (`await adapter.detect()`). Since file system IO operations are independent, this results in O(n) delay (where n is the number of adapters).
**Action:** Parallelize independent file system and adapter I/O operations (e.g., config reads/writes, adapter detections) using `Promise.all` instead of sequential `await` loops. When parallelizing operations where errors shouldn't halt the process (e.g., reading multiple configs), handle errors using `Promise.all` with an internal `try/catch` returning context alongside the error. Remember that `Promise.all` inherently preserves the order of the input array.
