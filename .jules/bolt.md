## 2024-03-08 - Parallel Promise Execution Requires Careful Iteration Ordering
**Learning:** When refactoring sequential loops that populate a configuration object based on system properties (like adapter detection iterating over `Object.entries(adapters)`), mapping the operations into an array of async tasks via `Promise.all` maintains the insertion order of the input array. However, we must iterate over the structured results of `Promise.all` directly instead of matching them asynchronously to prevent the progress spinner updating non-deterministically.
**Action:** Always map sequential async operations into an intermediate array of objects representing the context (`{ id, adapter, detected }`), resolve them via `Promise.all`, and then iterate synchronously over the structured result.
