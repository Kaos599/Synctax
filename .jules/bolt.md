## 2024-05-18 - Parallelized Sequential File Reads in Sync Command
**Learning:** Sequential adapter configuration reads (using `await adapter.read()`) during the diff preview and snapshot collection phases of `syncCommandInner` created unnecessary blocking operations, significantly slowing down the CLI execution when multiple clients are enabled.
**Action:** Always parallelize independent asynchronous operations like file I/O or network requests using `Promise.all`. Wrap individual operations in a `try/catch` block that returns a discriminated union (e.g., `{ success: true as const, data } | { success: false as const, error }`) instead of `Promise.allSettled` to reliably preserve and handle errors while maintaining access to original contextual objects (like the `adapter` instance).
