## 2024-05-11 - Single-pass reduce over multiple array filters
**Learning:** In scenarios where multiple aggregated counts (like success, partial, failed totals) are derived from the same result array (e.g., in backup manifestations and sync result summaries), using consecutive `.filter().length` calls iterates over the entire array multiple times. While array sizes here are not massive, standardizing to a single-pass `.reduce()` guarantees O(N) complexity and significantly minimizes array allocations, improving overall efficiency when iterating through CLI operation statuses.
**Action:** When calculating status counts or multiple aggregated totals from result arrays, prefer a single-pass `.reduce()` to avoid repeated O(N) iterations and unnecessary intermediate array creation.
