## 2025-02-18 - Avoid redundant sequential I/O in parallelizable tasks
**Learning:** In commands like `sync` that read data from multiple adapters to produce diffs and snapshots, performing those reads sequentially or repeating reads can create noticeable performance bottlenecks. Adapter I/O operations are independent and can be executed in parallel using `Promise.all`. However, handling errors requires careful structured return types (like `success: true as const` / `success: false as const`) instead of `Promise.allSettled`, so that explicit type tracking of failures works effectively.
**Action:** When working on operations that involve looping over multiple file/adapter reads, always look to see if the loops can be merged and the operations parallelized via `Promise.all`. If some could fail independently, return objects containing success booleans and the item to avoid throwing errors that fail the whole batch, while maintaining correct typing.
