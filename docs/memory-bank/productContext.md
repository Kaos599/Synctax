# Product Context

## Why this project exists
To eliminate the manual "sync tax" agentic developers pay when maintaining an ecosystem of AI tools.

## Problems it solves
1. Configuration Explosion (different JSON schemas, keys, locations).
2. Configuration Drift (inconsistent project rules and behaviors across tools).
3. Onboarding Friction (new teammates spend hours copying files).

## User Experience Goals
- A single `synctax` master command to manage everything safely.
- High visibility into sync drift with `synctax status`.
- Profile switching for work/personal contexts natively supported.
